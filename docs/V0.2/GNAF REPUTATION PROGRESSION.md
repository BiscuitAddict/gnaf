GNAF — SKILL \& REPUTATION PROGRESSION (TASK-BASED SCORING)



1\. CORE MODEL



\- Skill progression is earned through completed tasks

\- Each task contributes measurable skill growth

\- Growth is automatic and system-calculated





2\. SKILL CATEGORIES



Each member has domain-based skill tracks:



\- Execution Skill

\- Reliability Skill

\- Technical Skill

\- Collaboration Skill

\- Compliance Skill

\- Quality Skill





3\. TASK-BASED SCORING RULE



For every completed task:



\- Skill gain is calculated based on:

&#x20; • task complexity

&#x20; • completion quality

&#x20; • timeliness

&#x20; • verification outcome



\- Formula is system-defined and consistent across all guilds





4\. SKILL GAIN MECHANISM



\- Submission → base experience gain

\- Verification → quality multiplier applied

\- Hold period completion → trust stability bonus



Example flow:

\- Complete task → +XP

\- Verified task → +XP multiplier

\- Successful settlement → final XP lock-in





5\. NO MANUAL SKILL EDITING



\- Guild Owners cannot directly edit skills

\- Federation cannot manually boost individuals

\- Only completed work generates progression





6\. SKILL TRANSPARENCY MODEL



\- Members can see:

&#x20; • their own full skill breakdown

&#x20; • progress over time



\- Guild Owners can see:

&#x20; • member skill profiles for evaluation

&#x20; • historical growth patterns





7\. SYSTEM PHILOSOPHY



\- Skill = proof of execution history

\- Progress = accumulated real work, not titles or time served

\- Capability is emergent from repeated validated action
